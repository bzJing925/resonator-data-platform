除了前端 npm run dev 和后端 uvicorn 之外，必须单独启动 Celery worker，三者同时运行：

# 终端 1：后端 API
uv run uvicorn app.main:app --reload --port 8000

# 终端 2：任务队列 worker
uv run celery -A app.workers worker --loglevel=info --concurrency=4
生产环境用 bootstrap.sh up 会自动启动 worker，本地开发需要手动开。

如果后面上传还遇到任务卡住，先看 worker 是否在跑：

pgrep -af celery




./dev.sh —— 一键启动四个进程
./dev-down.sh —— 一键停止

本地开发必须同时保持四个进程运行：


# 1. Redis
redis-server --daemonize yes --port 6379

# 2. 后端 API
cd backend
/Users/jingbozuo/.local/bin/uv run uvicorn app.main:app --reload --port 8000

# 3. Celery worker
cd backend
/Users/jingbozuo/.local/bin/uv run celery -A app.workers worker --loglevel=info --concurrency=4

# 4. 前端
cd frontend
npm run dev
（你当前环境 uv 不在 PATH 里，所以要用 /Users/jingbozuo/.local/bin/uv 这个绝对路径，或者把 ~/.local/bin 加进 PATH。）

worker 日志里还会看到一些旧的 aln.process_batch 测试任务报错，这些不用管，它们失败后会从队列里清掉，不会影响新上传。

# 桌面单机版与自动更新设计

## 背景

当前 ALN 数据平台以 Web 应用为主：

- 浏览器 + Nginx + FastAPI + PostgreSQL + Redis + Celery 部署。
- 开发环境需要同时启动 Vite、Uvicorn、Celery worker，前后端容易断线。
- 已有 Electron + PyInstaller 桌面打包脚本（`build.py` / `backend/build_backend.py`），但运行时仍依赖 PostgreSQL/Redis，用户需要手动准备数据库，且没有自动更新能力。

用户希望：

1. 前端能以"不依赖端口的联网软件形式"运行——即用户双击安装包即可使用，不需要关心端口、数据库、消息队列。
2. 上传新版本后，软件能自动检测并提示更新。
3. 解决开发环境前后端经常断线的问题。

## 目标

**Phase 1**（本设计）聚焦现有 Electron 桌面版的增强：

- 桌面版完全单机零依赖运行（不需要 Docker / PostgreSQL / Redis）。
- Electron 自动管理后端生命周期、动态端口、后端保活/自动重启。
- 集成自动更新（GitHub Releases 优先，预留内网源接口）。
- 前端从 JavaScript 渐进迁移到 TypeScript。
- 改善开发环境：统一启动脚本、SSE/API 自动重连。

**Phase 2**（不在本设计范围内）再迁移到 Tauri + 云端推理/同步架构。

## 核心设计决策

1. **Electron 管理后端生命周期**：启动时动态发现端口、拉起 PyInstaller 后端、health 轮询、崩溃自动重启、退出时清理子进程。
2. **后端双模式运行**：同一套 FastAPI 代码通过 `DESKTOP_MODE=true` 切换为 SQLite + 本地队列；服务器版保持 PostgreSQL + Redis + Celery 不变。
3. **自动更新源可插拔**：默认 GitHub Releases，通过 `settings.json` 可切换为内网静态服务器。
4. **前端 TypeScript 渐进迁移**：先加类型、改扩展名、再关 `allowJs`，避免一次性大改引入 bug。
5. **开发环境改进**：统一启动脚本 + SSE/API 重连，减少日常开发断线。

## Electron 主进程

### 文件组织

```
frontend/
├── electron/
│   ├── main.ts          # 主进程入口
│   ├── preload.ts       # 预加载脚本
│   └── updater.ts       # 自动更新封装
├── src/                 # React TypeScript 源码
└── package.json
```

### 生命周期

| 阶段 | 行为 |
|---|---|
| 启动前 | 读取 `userData/settings.json`，决定更新源 |
| 端口发现 | 使用 Node.js 原生方法或 `get-port` 找一个空闲端口 |
| 启动后端 | 启动 `resources/backend/aln-backend`，传入 `DESKTOP_MODE=true PORT=<port> DATABASE_DIR=<userData>` |
| 健康等待 | 轮询 `http://127.0.0.1:<port>/api/health`，最多 60s，失败重试 3 次 |
| 创建窗口 | health 通过后加载前端页面 |
| 运行期 | 每 10s 检查 health；后端无响应则杀掉重启 |
| 退出 | `before-quit` 时发送 SIGTERM，5s 后 SIGKILL，清理子进程与临时文件 |

### 暴露给前端的 API

```ts
interface Window {
  aln: {
    getBackendUrl(): string;
    onBackendStateChange(cb: (state: 'starting' | 'ready' | 'error') => void): void;
    checkForUpdates(): Promise<{ version: string; available: boolean }>;
    installUpdate(): Promise<void>;
    openLogsFolder(): Promise<void>;
  };
}
```

### 错误处理

- 后端启动失败：窗口显示错误页与日志路径，提供"重试"按钮。
- 后端运行期崩溃：Electron 自动重启，前端进入"服务恢复中"遮罩，恢复后自动刷新。
- 端口冲突：动态端口避免该问题。

## 后端桌面模式

### 启动分支

```python
# app/config.py 伪代码
if os.getenv("DESKTOP_MODE") == "true":
    DATABASE_URL = f"sqlite:///{database_dir}/aln-data.db"
    TASK_BACKEND = "local"
else:
    DATABASE_URL = settings.DATABASE_URL
    TASK_BACKEND = "celery"
```

### SQLite 兼容策略

| PostgreSQL 特性 | SQLite 替代方案 |
|---|---|
| `timestamptz` | `DateTime(timezone=True)`，存储 UTC |
| `bigint generated by default as identity` | `BigInteger` + `autoincrement=True`（主键） |
| `ARRAY` 类型 | 改为 JSON 字符串或关联表 |
| 部分索引 | SQLite 3.8+ 原生支持 |
| Alembic 迁移 | 桌面模式使用 `Base.metadata.create_all()` + 版本表；服务器版继续用 Alembic |

### 本地任务队列

- 新增 `app/workers/local_worker.py`。
- 后台工作线程轮询 SQLite `tasks` 表，执行 `process_batch`。
- 进度写入 `tasks.progress` JSON 字段；前端 SSE 轮询该表获取进度。
- 支持任务取消：将任务状态改为 `CANCELLED`，worker 检查到后中断。

### 文件存储

桌面模式文件存储路径：`DATABASE_DIR/files/<batch_no>/...`。

## 自动更新机制

### 更新源抽象

```ts
type UpdateSource =
  | { type: 'github'; owner: string; repo: string }
  | { type: 'static'; url: string; channel: string };
```

### 流程

1. 应用启动 10s 后异步检查更新；用户也可手动检查。
2. 发现新版本后后台下载，向前端推送下载进度。
3. 下载完成提示用户重启安装。
4. 用户确认后调用 `autoUpdater.quitAndInstall()`。
5. 更新检查失败时记录日志，不阻塞使用。

### 版本与发布

- 版本号与 `frontend/package.json` 的 `version` 保持一致。
- GitHub Release tag 格式 `v0.2.0`，上传各平台安装包 + `latest.yml`。

## 前端 TypeScript 迁移

### 策略

1. 添加 `typescript`、React 类型、Vite 类型支持，`tsconfig.json` 设 `allowJs: true`。
2. 先迁移公共模块：`api/endpoints.js` → `api/endpoints.ts`，`hooks/useFields.js` → `useFields.ts`，`contexts/*.jsx` → `*.tsx`。
3. 再逐个迁移页面组件 `pages/*.jsx` → `*.tsx`。
4. 全部迁移后关闭 `allowJs`，删除旧 JS 文件。

### 类型入口

新增 `src/types/index.ts`，定义 `Device`、`Batch`、`Mapping` 等核心类型。

### 风险规避

- 迁移期间不改业务逻辑，只加类型注解。
- 每个文件迁移后跑 `npm run build` 验证。
- 使用 `git mv` 保留文件历史。

## 开发环境稳定性改进

### 统一启动脚本

在仓库根目录新增：

- `scripts/dev-start.sh`（macOS/Linux）
- `scripts/dev-start.ps1`（Windows）

同时启动：

1. Uvicorn（auto-reload）
2. Vite dev server
3. Celery worker

任一进程退出时可选全部退出或自动重启。

### 前端重连

- SSE 上传进度增加 `onerror` 自动重连（指数退避，最大 30s）。
- API 请求封装增加 `axios-retry`，对 502/503 重试 3 次。
- 后端未就绪时显示"服务启动中"，而不是报错。

## 验证

1. 运行 `python build.py` 生成桌面安装包，双击启动后无需手动配置端口/数据库。
2. 上传 zip 文件并等待处理完成，验证 SQLite 中生成 devices 数据。
3. 模拟后端进程崩溃，验证 Electron 自动重启并恢复页面。
4. 发布一个测试 GitHub Release，验证旧版本检测并安装更新。
5. 开发环境运行统一脚本，验证前后端断线后自动重连。
6. 运行 `npm run build` 与 `tsc --noEmit` 无类型错误。
